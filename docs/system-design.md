# 龙头复盘系统设计稿 v1.0

## 1. 系统目标

这不是一个“自动生成一篇复盘文章”的脚本，而是一套围绕龙头战法日常交易决策的复盘系统。

系统核心目标：

1. 每天自动拉取盘后数据，形成标准化市场面板。
2. 自动输出剔除 `ST` 后的短线关键指标。
3. 把“市场情绪 -> 主线题材 -> 龙头梯队 -> 核心个股 -> 明日预案”串成一个完整工作流。
4. 支持人工补充主观判断，不让系统只停留在机械统计。
5. 持续沉淀历史复盘，支持横向对比和复盘追踪。
6. 不只复盘单日，还要支持阶段复盘、周期回看、风格切换复盘。

一句话定义：

> 龙头复盘系统 = 数据引擎 + 情绪分析面板 + 主线龙头工作台 + 预案与复盘档案库

---

## 2. 用户角色

当前默认单用户设计，后续可扩展多用户。

主要用户：

1. 个人短线交易者
2. 龙头战法研究者
3. 量化策略迭代者

当前优先级最高的场景：

1. 收盘后 30 分钟内快速完成复盘
2. 晚上整理主线、龙头、预案
3. 次日竞价前快速回看昨日结论
4. 周末回看最近 5 个交易日、10 个交易日、20 个交易日的演化
5. 在一段时间内横向比较不同主线、不同龙头、不同情绪阶段

---

## 3. 系统边界

### 3.1 系统负责的内容

1. 数据抓取
2. 指标计算
3. 复盘模板生成
4. 主线/龙头候选排序
5. 复盘记录保存
6. 历史对比与统计

### 3.2 系统不完全自动化的内容

1. 新闻语义理解
2. 题材命名
3. 真正的“市场共识龙头”最终确认
4. 分时盘口细节判断
5. 尾盘情绪主观修正

所以这个系统必须设计成：

- `自动计算 + 人工确认`

而不是：

- `全自动给结论`

---

## 4. 核心业务流

### 4.1 日终流程

1. 收盘后拉取全市场日线、涨跌停、指数、行业、概念数据
2. 剔除 `ST`
3. 计算市场情绪指标
4. 统计主线题材强度
5. 识别龙头梯队与核心票
6. 生成“今日复盘初稿”
7. 用户人工补充：
   - 主线判定
   - 真龙头
   - 今日买点/卖点
   - 明日预案
8. 保存为正式复盘

### 4.2 次日盘前流程

1. 读取最近 1-3 日复盘
2. 展示昨日主线、昨日龙头、昨日结论
3. 展示今日竞价前观察清单
4. 生成竞价预案卡片

### 4.3 周期复盘流程

1. 选择复盘周期：
   - 最近 5 个交易日
   - 最近 10 个交易日
   - 最近 20 个交易日
   - 自定义区间
2. 汇总周期内每日市场情绪、主线、龙头、风险锚
3. 统计主线持续性、龙头切换次数、接力环境变化
4. 对比周期前半段和后半段的风格变化
5. 输出“阶段总结 + 可视化对比 + 下阶段预案”

### 4.4 横向对比流程

1. 在多个交易日之间横向比较：
   - 情绪强弱
   - 主线强度
   - 龙头高度
   - 赚钱效应 / 亏钱效应
2. 支持按主题查看：
   - 某一条主线在一段时间内的演化
   - 某只龙头股在一段时间内的地位变化
   - 某种情绪阶段出现时，后续市场如何演变
3. 输出结论：
   - 当前市场更像哪一段历史
   - 当前应对策略更接近进攻还是防守

## 5. 系统模块拆分

建议拆成 6 个模块。

### 5.1 数据引擎模块

职责：

1. 接入 `Tushare`
2. 抓取交易日、指数、个股日线、涨跌停、行业、概念、龙虎榜等
3. 做基础清洗和缓存

输出：

1. 原始市场数据表
2. 标准化个股快照表
3. 交易日历表

### 5.2 市场情绪模块

职责：

1. 计算市场宽度
2. 计算涨停/跌停/连板/晋级/溢价
3. 给市场情绪打分
4. 输出市场阶段结论

核心指标：

1. 上涨家数
2. 下跌家数
3. 涨停家数（剔除ST）
4. 跌停家数（剔除ST）
5. 炸板家数
6. 昨日涨停溢价
7. 昨日连板晋级率
8. 非ST最高连板
9. 非ST连板梯队
10. 两市成交额及变化

输出：

1. `emotion_score`
2. `market_stage`
3. `risk_level`

### 5.3 主线题材模块

职责：

1. 统计行业/概念活跃度
2. 判断是否具备主线特征
3. 给题材分级：主线 / 次主线 / 轮动 / 放弃

核心指标：

1. 题材涨停家数
2. 题材连板家数
3. 题材前排强度
4. 题材成交额
5. 题材持续天数
6. 题材核心股辨识度

输出：

1. `theme_rank`
2. `main_theme`
3. `secondary_theme`

### 5.4 龙头梯队模块

职责：

1. 识别市场高度板
2. 识别题材龙头
3. 识别中军、补涨、跟风

核心指标：

1. 连板数
2. 成交额
3. 所属题材
4. 连续强势天数
5. 次日溢价
6. 在题材中的领先性

输出：

1. `market_leader`
2. `theme_leaders`
3. `capacity_core`
4. `risk_anchor`

### 5.5 复盘工作台模块

职责：

1. 生成复盘初稿
2. 让用户人工修订
3. 保存正式结论

需要支持的输入：

1. 今日主线
2. 真龙头
3. 最佳买点
4. 最差交易
5. 明日预案
6. 仓位计划

输出：

1. 每日正式复盘文档
2. 明日盘前观察清单

### 5.6 历史档案模块

职责：

1. 保存每天的复盘结果
2. 做历史对比
3. 追踪主线持续性和龙头切换
4. 支持阶段复盘与区间复盘
5. 支持多日横向比较

支持查询：

1. 某题材近 20 天出现次数
2. 某只股票被标记为龙头的次数
3. 某种情绪阶段下的交易表现
4. 某类买点成功率
5. 最近 5/10/20 天市场情绪曲线
6. 最近 5/10/20 天主线切换路径
7. 最近 5/10/20 天龙头高度变化
8. 不同时间段的接力环境对比

### 5.7 周期复盘模块

职责：

1. 把单日复盘汇总成阶段复盘
2. 识别某段时间的市场主旋律
3. 总结主线持续性、龙头切换、风格漂移
4. 输出“本周期最值得记住的规律”

周期维度：

1. `5日`
2. `10日`
3. `20日`
4. `月度`
5. `自定义区间`

核心输出：

1. 周期市场阶段
2. 周期主线强度排名
3. 周期龙头排行榜
4. 周期风险锚排行榜
5. 周期赚钱效应最强买点
6. 周期亏钱效应最强错误模式

### 5.8 横向对比模块

职责：

1. 比较多个交易日之间的异同
2. 支持同周期内分段比较
3. 支持不同周期横向比较

比较维度：

1. 情绪分对比
2. 成交额对比
3. 主线对比
4. 龙头高度对比
5. 涨停/跌停对比
6. 晋级率对比
7. 昨日涨停溢价对比
8. 赚钱效应方向对比
9. 亏钱效应方向对比

## 8. 关键算法设计

### 8.1 情绪分算法

建议结构：

1. 涨停数加分
2. 跌停数减分
3. 最高板加分
4. 非ST连板梯队加分
5. 昨日涨停溢价加分或减分
6. 昨日晋级率加分或减分
7. 缩量或放量做微调

输出：

- `0-100`

分区：

1. `0-30`：退潮
2. `31-50`：弱分化
3. `51-70`：中性轮动
4. `71-85`：修复转强
5. `86-100`：主升强势

### 8.2 主线评分算法

建议结构：

1. 涨停家数权重 `30%`
2. 连板数量权重 `20%`
3. 板块成交额权重 `15%`
4. 龙头辨识度权重 `20%`
5. 持续天数权重 `15%`

### 8.3 龙头评分算法

建议结构：

1. 连板高度
2. 题材地位
3. 成交额
4. 是否容量核心
5. 近期强度
6. 次日溢价表现

输出角色：

1. `总龙头`
2. `题材龙头`
3. `中军`
4. `补涨`
5. `跟风`
6. `风险锚`

### 8.4 周期复盘算法

建议结构：

1. 统计周期内情绪均值、最高值、最低值
2. 统计周期内主线出现频次
3. 统计周期内龙头切换次数
4. 统计周期内非ST最高连板变化
5. 统计周期内昨日涨停溢价均值
6. 统计周期内晋级率均值

周期结论建议输出：

1. 本周期是主升、轮动、修复还是退潮
2. 本周期最强主线是什么
3. 本周期主线是否持续还是频繁轮动
4. 本周期高标是否有赚钱效应
5. 本周期最值得做的交易模式是什么
6. 本周期最危险的错误模式是什么

### 8.5 横向对比算法

建议结构：

1. 对比最近 N 天每日情绪分
2. 对比最近 N 天每日主线是否切换
3. 对比最近 N 天每日龙头高度变化
4. 对比最近 N 天每日涨停/跌停变化
5. 对比最近 N 天昨日涨停溢价变化
6. 标注“拐点日”

拐点日识别可先用这些条件：

1. 情绪分变化超过 `15`
2. 涨停家数变化超过 `20`
3. 昨日涨停溢价由负转正或由正转负
4. 主线切换
5. 龙头高度断层

## 12. 最终产品定义

最终你要的不是一个脚本，而是一个每天都能用的交易工作台：

1. 收盘后自动出数据
2. 自动标出市场情绪和主线候选
3. 自动列出龙头梯队和风险锚
4. 你补一句话判断
5. 系统自动归档
6. 次日开盘前直接回看
7. 周末可以回看最近一周、两周、一个月的市场演化
8. 能对比多个时间段，找到行情切换和策略适配点

一句话总结：

> 龙头复盘系统的核心，不是“写文章”，而是“建立一套可重复执行的交易认知工作流”。

再补一句：

> 真正有价值的龙头复盘系统，一定不只是“今天发生了什么”，而是“这一段时间市场是如何变化的，以及这种变化意味着什么”。
